Which files this dictionary covers

The pipeline writes one set of summary files inside the TOML folder under postprocessing/. These are the files most likely to be concatenated across many videos, cameras, days, acts, and arena cells.

  • postprocessing/ba_postprocessing/ba_individual_summary_all.csv: one row per BA animal/session/cell. BA has one beetle per TOML.
  • postprocessing/fight_postprocessing/fight_individual_summary_all.csv: one row per beetle in each fight cell. Fight TOMLs have two animals, so each cell usually contributes two rows.
  • postprocessing/fight_postprocessing/fight_pair_summary_all.csv: one row per fight cell/session summarizing pairwise distance, contact, and possible fight-like close-contact events.

The package may also write metadata-tagged copies of these files, such as <metadata_tag>_fight_pair_summary_all.csv. Those tagged copies contain the same columns as the generic files. The tag is included so the file remains interpretable if copied away from the TOML folder.

Recommended analysis practice: Use the generic summary files in scripted workflows, but preserve the metadata-tagged copies when manually moving files. Either way, every row contains source-video and TOML provenance columns.

Interpretation notes

Frames, seconds, and timelapse videos

Columns ending in _frames are frame counts. Columns ending in _s are seconds computed from the fps value supplied to post-processing. If your biological sampling is effectively one frame per second, then 300 frames is roughly 300 seconds of biological time. Do not interpret frame counts as 30 fps video time unless the post-processing fps parameter was intentionally set that way for the analysis.

Movement onset metrics

The displacement-threshold latency columns ask when the animal first moved more than move_threshold_px away from its starting position. The sustained threshold is stricter because the animal must stay beyond threshold for movement_onset_consecutive_frames.

Contact versus possible fight-like events

contact_px is a broader proximity threshold. fight_px is a stricter very-close-contact threshold. The script labels these as possible fight-like events because centroid proximity alone cannot prove a specific aggressive behavior without validation against scored video.

Turtling-like events

Turtling-like events are centroid-pattern detections, not direct posture classifications. They are intended to flag periods of compact, looping, low-net-displacement movement after movement onset. Treat them as candidate events that should be validated against video examples.

Missing values and quality control

  • NaN means the value could not be computed. Common reasons include no valid positions, no threshold crossing, no secondary ROI, or no event of that class.
  • High fraction_total_interpolated_position_frames means a large fraction of the trajectory was reconstructed rather than directly tracked. Review maps and session quality before trusting biological conclusions.
  • Low fraction_valid_pair_frames in fight summaries means pairwise distance metrics were computed from a limited subset of frames.
  • The safest row identifiers for joining and filtering are original_video_stem, cell_label, toml_name, output_stem, and animal_index_0_based for individual summaries.
