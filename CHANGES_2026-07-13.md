# Changes: 2026-07-13

- Added direct selection of one or multiple TOML files from the GUI.
- Selected-file mode is preserved during validation refresh and manifest creation.
- Added extended table row selection using Shift-click and Ctrl/Command-click.
- Include/exclude actions now operate on all selected rows at once.
- Added `install_process_tomls_command.sh` to install `process_tomls` inside the active conda environment.
- Updated `install_on_firebird.sh` and README instructions.

## Explicit CPU post-processing-only button

- Added **Run post-processing only (CPU)** beside the full-pipeline button.
- This path rebuilds the manifest from the TOMLs currently included in the GUI.
- It submits `MODE=postprocess` directly to SLURM and does not submit IDtracker.ai or session-collection jobs.
- It requests no GPU and uses the optional Post-processing partition field, or the cluster default partition when that field is blank.
- It requires existing collected sessions under `<TOML folder>/idtracker_sessions` and stops before submission if that directory is missing.
- The existing full pipeline remains available as **Run full pipeline (IDtracker + post-processing)**.
