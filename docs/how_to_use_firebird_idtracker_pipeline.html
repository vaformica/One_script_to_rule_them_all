Firebird IDtracker.ai TOML Folder Pipeline

This package organizes IDtracker.ai runs where one source video contains one or more arenas/cells, and each arena/cell has its own TOML created manually in the idtracker.ai Segmentation app.

Known-good design: the workflow uses the three-stage structure first stabilized in v8: GPU tracking array, CPU session collector, CPU post-processing.

What students do

  1. Open the idtracker.ai Segmentation app and create TOMLs by hand for one video.
  2. Put all TOMLs for that video into one TOML folder.
  3. Open this GUI on Firebird noVNC.
  4. Select the original video, the TOML folder, and BA or Fight.
  5. Import and validate the TOML folder.
  6. Edit cell labels if needed.
  7. Run the SLURM pipeline.

Install and launch

conda activate idtrackerai
python -m zipfile -e IDTracker_Firebird_TOML_Folder_Package_v12_final.zip .
cd IDTracker_Firebird_TOML_Folder_Package
bash install_on_firebird.sh
cd ~/formicalab/IDTracker_Firebird_TOML_Folder
bash launch_gui.sh

Video/TOML mismatch protection

When TOMLs are imported, the package compares the source video selected in the GUI against the video path/name recorded inside each TOML. If a hard mismatch is detected, the run is blocked and the GUI shows a clear error. The safest response is to clear the selection and choose the correct video and TOML folder.

Outputs

Outputs are linked to the TOML folder. Session folders are moved out of the original video folder and into idtracker_sessions/. Post-processing files go under postprocessing/ba_postprocessing/ or postprocessing/fight_postprocessing/.

The final summary files include provenance columns and are also copied with the metadata tag in the filename, so they remain identifiable if moved elsewhere.

Standalone post-processing

Fight post-processing without the GUI:

conda activate idtrackerai
cd ~/formicalab/IDTracker_Firebird_TOML_Folder
python standalone/run_fight_postprocessing_standalone.py --toml-folder /path/to/TOML_FOLDER

BA post-processing without the GUI:

conda activate idtrackerai
cd ~/formicalab/IDTracker_Firebird_TOML_Folder
python standalone/run_ba_postprocessing_standalone.py --toml-folder /path/to/TOML_FOLDER

Submit standalone post-processing to CPU SLURM:

python standalone/run_fight_postprocessing_standalone.py --toml-folder /path/to/TOML_FOLDER --submit-cpu --cpu-partition ""

Successful status

Trust the run when status says Overall status: COMPLETE, Collected sessions with trajectories: N/N, and Post-processed cells OK: N/N.
