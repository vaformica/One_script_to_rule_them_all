Firebird IDtracker.ai TOML-Folder Pipeline

Student guide, data dictionary, and parameter reference

This guide explains how to process IDtracker.ai TOML files on Firebird for BA/single-beetle assays and fight/two-beetle assays.

1. What this pipeline does

This package is a Firebird-friendly wrapper around IDtracker.ai and the lab's post-processing scripts. It is designed for the common lab workflow where a single original video contains one or more separate arenas or cells. Students use the IDtracker.ai Segmentation app by hand to create one TOML file per arena/cell, then this pipeline runs all the TOMLs in batch, organizes the IDtracker.ai sessions, and creates analysis CSVs and QC maps.

The basic idea: the original video is the raw data, each TOML is one arena/cell from that video, and every output row is tagged with the original video name, TOML name, cell label, and metadata tag so results remain traceable after CSV files are concatenated.

BA pipeline

Use for one-beetle behavioral assay videos. Expected animals per TOML: 1. Outputs individual movement, onset, ROI/wall-buffer, interpolation, turtling-like QC, per-frame kinematics, and track maps.

Fight pipeline

Use for two-beetle combat videos. Expected animals per TOML: 2. Outputs BA-style metrics for both beetles plus pairwise distance, contact/proximity events, possible fight-like events, InqScribe text, and track maps.

2. Key concepts and vocabulary

TermMeaningWhy it matters
Original videoThe source .mp4, .avi, or similar file used in IDtracker.ai.This is the primary raw-data identity. All output rows should point back to this video.
TOML fileIDtracker.ai parameter file created manually in the Segmentation app.Each TOML stores the selected video, ROI polygon, tracking interval, number of animals, and segmentation settings.
Cell / arena labelThe arena identifier, usually from the end of the TOML filename, such as A1, B3, or C2.The same video can have multiple TOMLs. The cell label keeps each arena separate.
Metadata tagHuman-readable tag entered in the GUI, often the video stem or experiment label.This is copied into summary CSV filenames and every row of output data.
ManifestCSV created by the GUI that links video → TOML → cell label → output stem.The manifest is the authoritative map between raw video, TOML, session folder, and output files.
IDtracker sessionFolder created by IDtracker.ai containing trajectory arrays and session metadata.The pipeline moves full session folders into the TOML folder so video storage stays clean.
CollectorPipeline stage that finds completed IDtracker sessions and moves them to the organized output folder.This stage prevents IDtracker.ai session folders from being left in the original video folder.
Post-processingPython analysis stage after tracking.Creates summaries, events, per-frame data, maps, and data dictionaries.

3. Setup on Firebird

Use the idtrackerai conda environment when launching the GUI. The normal workflow is to run the GUI from noVNC on Firebird.

Install the package

conda activate idtrackerai
python -m zipfile -e IDTracker_Firebird_TOML_Folder_Package_v13_partition_default_fix.zip .
cd IDTracker_Firebird_TOML_Folder_Package
bash install_on_firebird.sh

Launch the GUI

conda activate idtrackerai
cd ~/formicalab/IDTracker_Firebird_TOML_Folder
bash launch_gui.sh

Do not launch from the base environment. If the terminal prompt says (base), run conda activate idtrackerai first. The GUI and IDtracker.ai need packages from that environment.

4. Student workflow from video to output CSVs

Create TOMLs manually in IDtracker.ai

Open the IDtracker.ai Segmentation app. For each arena/cell in the video, set the ROI polygon, blob size/background subtraction settings, tracking interval, and expected number of animals. Save one TOML per arena/cell.

Recommended naming pattern:

Camera_1_40169154_20260630_1342_FIGHT_ACT1_A1.toml
Camera_1_40169154_20260630_1342_FIGHT_ACT1_A2.toml
Camera_1_40169154_20260630_1342_FIGHT_ACT1_B1.toml

Put TOMLs for one video in one folder

One TOML folder should contain TOMLs for one original video and one assay type only. Do not mix fight and BA TOMLs in the same folder. Do not mix TOMLs from different videos.

Select the original video and TOML folder in the GUI

Choose the original video file first, then the TOML folder. Choose the pipeline: BA or Fight. Each video should be either BA or fight, not both.

Import and validate

Click the import/validate button. The GUI reads all TOMLs, checks that they point to the selected video, extracts the tracking interval start frame, checks expected animal number, and creates the import grid.

If the video and TOMLs do not match, stop and choose the correct video or correct TOML folder. The safest consequence is to clear the current selection and start the import again. The GUI should not delete TOMLs.

Check cell labels

The GUI guesses the cell label from the end of each TOML filename, such as A1 or B3. Edit labels if needed. Cell labels must be unique within a video.

Remove TOMLs from the run if needed

Use Remove selected TOML from run to exclude a TOML from the current manifest without deleting the file. Use Restore selected TOML to run to include it again.

This is useful when most cells already ran successfully but one TOML needs to be rerun or reprocessed.

Run the pipeline

For a full run, the GUI submits three stages to SLURM:

  1. IDtracker.ai GPU array job.
  2. Session collector job.
  3. Post-processing job.

The default GPU request is Any GPU to maximize the number of simultaneous jobs. L40S-only is available but may run fewer jobs at once.

5. Reruns, partial runs, and overwriting

SituationRecommended actionWhat happens
First time running a TOML folderFull pipelineRuns IDtracker.ai, collects sessions, runs post-processing.
Only maps/CSV outputs need updatingPython-only post-processingUses existing collected session folders and regenerates post-processing outputs.
One TOML was corrected in Segmentation appRemove successful TOMLs from run, keep corrected TOML active, run full pipeline for that subsetRuns only the active TOMLs in the manifest.
A TOML folder already has sessions and post-processing outputsUse the GUI prompt to choose rerun IDtracker, rerun Python only, or do nothingPrevents silent overwrites and makes reruns intentional.
A TOML file is wrong and should be recreatedRemove it from the run or fix it in IDtracker.aiThe GUI does not delete TOML files.

Important: Python-only work is still normally submitted to SLURM as a CPU/default-partition job, so it can keep running after the GUI is closed. If the CPU/default partition is busy, it may sit pending before it starts.

6. How to know whether the pipeline is finished

The GUI's status summary is the easiest place to check. A successful complete run should say:

  • Overall status: COMPLETE
  • Collected sessions with trajectories: N/N
  • Post-processed cells OK: N/N
  • Warnings: none
  • Errors: none

Useful SLURM commands

squeue -u vformic1-swat

To watch jobs update every 10 seconds:

watch -n 10 'squeue -u vformic1-swat'

To cancel old stuck jobs by ID:

scancel 823224 823119

Completion markers

FileMeaning
project_metadata/_SESSION_COLLECTION_COMPLETE_ALL_CELLS.txtCollector found and moved sessions for all active TOML rows.
project_metadata/_SESSION_COLLECTION_INCOMPLETE_OR_FAILED.txtCollector could not verify all expected sessions.
postprocessing/fight_postprocessing/_POSTPROCESS_COMPLETE_ALL_CELLS.txtFight post-processing succeeded for all active TOMLs.
postprocessing/ba_postprocessing/_POSTPROCESS_COMPLETE_ALL_CELLS.txtBA post-processing succeeded for all active TOMLs.
_POSTPROCESS_INCOMPLETE_OR_FAILED.txtAt least one cell did not post-process successfully.

Once jobs are submitted to SLURM, they continue running even if the GUI closes, noVNC disconnects, or your laptop sleeps. The GUI is a submitter/status viewer; SLURM owns the jobs.

9. Parameter reference

9.1 Shared parameters

GUI label / flagDefaultWhat it doesPractical advice
Analysis window length, frames
--window-frames		7500Number of frames analyzed after the TOML-derived analysis start frame.Increase for longer runs. Decrease for quick tests.
Video frames per second
--fps		30Used to convert frames to seconds in event files and convenience columns. Does not change the frame indices used for analysis.If one analyzed frame equals one biological second, use 1 for biologically meaningful seconds. If analyzing true 30 fps video, use 30.
Movement threshold, pixels
--move-threshold-px		30Displacement/cumulative movement threshold for movement onset metrics.Increase if small jitter is being counted as movement. Decrease if real movement is subtle.
Sustained movement frames
--movement-onset-consecutive-frames		30Frames required for sustained movement onset after crossing threshold.Increase to require more persistent movement. Decrease for more sensitive onset detection.
Artifact jump threshold, pixels
--max-step-px		30One-frame jumps larger than this are treated as likely tracking artifacts and interpolated if bounded by valid positions.Increase if fast real movement is falsely flagged. Decrease if identity/tracking jumps remain in the data.
ROI wall buffer, pixels
--roi-wall-buffer-px		30Distance from primary ROI boundary used to classify border/wall occupancy.Set based on arena size and how close to the wall counts as wall-associated behavior.
Track linewidth
--track-linewidth		0.4Line thickness on QC maps.Increase if maps are too faint. Decrease if dense paths are unreadable.
Map max overlay points
--map-max-overlay-points		400Maximum number of interpolation/turtling overlay points drawn on maps.Prevents maps from becoming unreadable when many frames are flagged.

9.2 Fight-specific parameters

ParameterDefaultMeaningPractical effect
--animal0, --animal10, 1Zero-based indices for the two animals in the trajectory file.Usually leave as 0 and 1.
--contact-px60Pairwise distance threshold for contact/proximity events.Lower is stricter. Higher captures broader proximity.
--min-contact-s0.2Minimum contact duration in seconds, converted using FPS.Use with caution if FPS does not equal biological sampling rate.
--fight-px35Stricter pairwise distance threshold for possible fight-like close-contact events.Lower means only very close episodes are flagged.
--min-fight-frames6Minimum frames required for possible fight-like event.Increase to remove brief close passes. Decrease to detect shorter events.

9.3 Turtling-like event parameters

Turtling output is not a posture classifier. It uses centroid path shape: enough path length, low net displacement, compact spatial spread, low straightness, and turning. Treat it as a QC/event-discovery measure that should be verified against video.

ParameterDefaultWhat stricter meansNotes
--turtling-window-frames300Larger window requires behavior to persist over a longer segment.In 1 fps biological time, 300 frames is about 300 seconds.
--turtling-min-duration-frames300Larger value requires longer retained events.Do not blindly set to 600 if each frame is one second unless you really mean 10 minutes.
--turtling-merge-gap-frames60Smaller value keeps nearby bouts separate.Larger value merges nearby candidate windows.
--turtling-max-net-displacement-px80Lower value requires the animal to end closer to where it started.Good control for avoiding ordinary travel being marked turtling-like.
--turtling-max-radius-gyration-px50Lower value requires spatially compact behavior.This is often a better strictness knob than duration.
--turtling-max-straightness0.25Lower value requires a less straight, more looping path.Straightness = net displacement / path length.
--turtling-min-path-px80Higher value requires more total movement within the compact area.Helps distinguish turtling-like scrambling from simply sitting still.
--turtling-min-abs-turn-rad0.30Higher value requires more turning.Useful if straight shuffling is being flagged.
--turtling-start-buffer-frames300Larger value ignores early frames after movement onset.Useful if early wake-up behavior is being mislabeled.

If turtling is too permissive, first try lowering turtling_max_radius_gyration_px, lowering turtling_max_net_displacement_px, increasing turtling_min_path_px, or increasing turtling_min_abs_turn_rad. Duration is biologically meaningful, so change it only if you really want longer events.

10. Running post-processing without the GUI

The package includes standalone scripts for cases where the GUI is unavailable or you only want to rerun Python post-processing.

Fight post-processing only

conda activate idtrackerai
cd ~/formicalab/IDTracker_Firebird_TOML_Folder
python standalone/run_fight_postprocessing_standalone.py \
  --toml-folder /path/to/TOML_FOLDER

BA post-processing only

conda activate idtrackerai
cd ~/formicalab/IDTracker_Firebird_TOML_Folder
python standalone/run_ba_postprocessing_standalone.py \
  --toml-folder /path/to/TOML_FOLDER

Submit standalone post-processing to SLURM

Leave the partition blank unless Firebird staff tell you the correct CPU partition name.

python standalone/run_fight_postprocessing_standalone.py \
  --toml-folder /path/to/TOML_FOLDER \
  --submit-cpu

11. Troubleshooting

SymptomLikely causeWhat to do
GUI fails with no PyQt/PySide moduleLaunched from base instead of idtrackerai, or Qt is not installed in environment.Run conda activate idtrackerai. If needed, install PyQt in that environment.
Video/TOML mismatch errorTOMLs point to a different video than the one selected in GUI.Clear the selection, choose the correct original video and TOML folder, then re-import.
SLURM says invalid partitionA partition name such as cpu was specified but does not exist on Firebird.Leave the post-processing partition field blank so SLURM uses the default valid partition.
IDtracker jobs finish but post-processing waitsCollector or post-processing job is pending in SLURM.Check squeue -u vformic1-swat. If pending for resources, wait or ask which CPU/default partition should be used.
Old jobs remain with DependencyNeverSatisfiedOld dependent jobs from a failed run.Cancel the specific job IDs with scancel JOBID.
Session folders appear beside original videosIDtracker.ai writes sessions near the original video before collector moves them.This is expected during/after tracking. The collector should move full sessions into the TOML folder.
Status says collected 0/N while tracking is still runningCollector has not run yet.Wait until IDtracker array finishes and collector starts.
Maps look wrongCould be legend layout, ROI parsing, coordinate scale, or old maps not overwritten.Run Python-only post-processing, check map timestamps, and inspect all_track_maps.

Diagnostic ZIP command

If something fails, create a lightweight diagnostic bundle from inside the TOML folder. This does not copy the large video or trajectory arrays.

DEBUG_DIR="idtracker_debug_$(date +%Y%m%d_%H%M%S)"
ZIP_NAME="${DEBUG_DIR}.zip"
mkdir -p "$DEBUG_DIR"
cp -r project_metadata "$DEBUG_DIR/" 2>/dev/null || true
cp -r logs "$DEBUG_DIR/" 2>/dev/null || true
find . -maxdepth 1 -name "*.toml" -exec cp {} "$DEBUG_DIR/" \;
find postprocessing -type f \( -name "*.csv" -o -name "*.txt" -o -name "*.json" -o -name "*.png" \) -exec cp --parents {} "$DEBUG_DIR/" \; 2>/dev/null || true
python -m zipfile -c "$ZIP_NAME" "$DEBUG_DIR"
ls -lh "$ZIP_NAME"
